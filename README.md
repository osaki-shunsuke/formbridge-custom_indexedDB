# formbridge-custom_indexedDB

Antigravity（AIエディター）にこれまでの経緯と現在のコードの意図を正確に伝えるための要約を作成しました。これをそのままチャット欄や指示書として渡せば、コンテキストを完璧に同期できるはずです。

Antigravityへの共有用サマリー
現在、kintone連携サービス「FormBridge」において、**「電波の悪い現場で1レコードあたり最大50枚の写真を確実に一時保存・送信する」**ためのカスタマイズを開発中です。
ここまでの実装内容と、解決済みの技術的課題、および現在の修正方針を共有します。

1. 開発の背景と制約
データ量: 1回あたり50枚の画像を扱うため、FormBridge標準の一時保存機能（localStorage / 5MB制限）は容量不足で採用せず、IndexedDBによる自作のバックアップ機構を採用。

画像処理: 送信時の負荷軽減のため、Compressor.js を用いてクライアントサイドで画像を圧縮（quality 0.7 / maxWidth 1280）。

2. 解決済みの重要課題
CSP（セキュリティ）回避: fetch() によるBase64解析がブラウザのセキュリティ（CSP）でブロックされる現象を、base64ToBlob という手作業によるバイナリ変換関数を自作することで突破。

Vue.jsの仮想DOM対策（不死身デーモン）: FormBridge（Vue.js）がDOMを書き換えて自作UI（緑のバナー等）を消去してしまう問題に対し、1.5秒間隔でUIを強制復元する setInterval （デーモンプログラム）を実装。

アップロード同期: オンライン復帰時に自動でアップロードが開始されない問題に対し、モード切替時に location.reload() による強制リロード を実行し、form.show 時の復元フローに乗せることで解決。

3. 現在取り組んでいる修正
送信ボタンの制御: オフラインモード時に誤って「画像が空のまま送信」されるのを防ぐため、回答ボタン（および確認ボタン）を非表示にする。

実装方針: ProgressIndicator オブジェクトに toggleSubmitButton メソッドを追加。特定のクラス名だけでなく、中身のテキスト（「回答」等）を検知して closest('button') で親要素ごと隠す、より堅牢なセレクタ制御を行う。

デーモンへの統合: 画面遷移などでボタンが復活しないよう、前述の1.5秒間隔のデーモン処理の中で常に非表示状態を監視・維持する。

4. 追加実装した堅牢性・UX向上の機能 (2026/03/09)
現場での運用をよりスムーズにし、トラブルを未然に防ぐため、以下の3つの機能を実装しました。
- 古い一時保存データの自動破棄 (Garbage Collection): 「送信完了後」だけでなく、保存から一定期間経過した古いデータが残っていた場合、復元確認を出さずに自動で破棄する機能を実装。コード内の `EXPIRE_DAYS` 変数で破棄までの期限（デフォルト: 7日間）を容易に変更できるように設計。（これにより端末容量の圧迫を防ぐ）
- ネットワーク復帰時の自動検知・通知アラート: ブラウザ標準の `online` イベントを監視し、圏外からオンラインに復帰した瞬間にアラートを表示。ユーザーにオフラインモードの解除と送信を促す。
- 事前ストレージ容量チェック (Storage API): `navigator.storage.estimate()` を使用してフォームを開いた時点で空き容量を計算。残り容量が50MB未満の場合に事前警告を出し、保存時のエラーを未然に防ぐ。

5. ロールバック手順（課題1〜3実装前に戻す場合）
実装した追加機能（課題1〜3）により不具合が発生した場合、以下のGitコマンドを実行することで、機能追加前の安定した状態（コミットハッシュ: `ee75193`）に戻すことができます。

```bash
# 変更を強制的に機能追加前の状態に戻す（※テスト等で行ったコミットされていない変更は消えるので注意）
git reset --hard ee75193

# もしGitHub（リモート）へも強制反映させる必要がある場合（※複数人開発の場合は注意）
git push -f origin main
```
